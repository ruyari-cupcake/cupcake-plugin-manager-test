# Cupcake Provider Manager 사용 설명서

> 배포용 사용자 문서
> 
> 이 문서는 **사용자가 실제로 무엇을 눌러야 하는지**와 **설정을 바꾸면 바로 어떤 느낌으로 반영되는지**만 설명합니다.

---

## 1. 먼저 알아둘 것

- 이 플러그인은 **저장 버튼이 없습니다.**
- 설정창에서 값을 바꾸면 **그 순간 바로 저장**됩니다.
- 따라서:
  - 값을 넣었다가 지우면 바로 반영됩니다.
  - 체크박스를 켜거나 끄면 바로 저장됩니다.
  - 실수로 바꿨다면 다시 원래 값으로 돌리면 됩니다.

---

## 2. 설치 순서

### 2-1. 메인 플러그인 설치

1. RisuAI에서 **설정 → 플러그인**으로 이동합니다.
2. **+ 버튼**으로 배포된 메인 플러그인 파일을 설치합니다.
3. 설치 후 CPM 설정창을 엽니다.

### 2-2. 서브 플러그인 설치

1. CPM 설정창에서 **🧩 Sub-Plugins** 탭으로 이동합니다.
2. **설치할 서브 플러그인 선택** 영역을 눌러 `.js` 또는 `.mjs` 파일을 선택합니다.
3. 배포받은 서브 플러그인들을 **하나씩 설치**합니다.
4. 설치가 끝나면 왼쪽 사이드바에 각 프로바이더 탭이 나타납니다.

### 2-3. 처음 설치 후 추천

처음 한 번은 아래 순서로 세팅하면 편합니다.

1. 사용할 프로바이더 탭에 들어가서 **키부터 입력**
2. 필요하면 **모델 목록 불러오기** 옵션 체크
3. RisuAI 메인 모델 선택창에서 사용할 모델 선택
4. 그 다음에 글로벌 기본값 / 슬롯 설정 조정

---

## 3. 가장 중요한 개념 3개

### A. 프로바이더 탭 설정
- 여기서는 주로 **API 키, 모델 목록 불러오기, 프로바이더별 추가 옵션**을 설정합니다.
- 보통 사용자는 **이 탭에서 인증과 공급자별 옵션**만 맞추면 됩니다.

### B. 글로벌 기본값
- **리스AI가 어떤 값을 안 보냈을 때만** 대신 들어가는 값입니다.
- 즉, 글로벌은 **강제 덮어쓰기용이 아니라 빈칸 보충용**입니다.

### C. 슬롯 설정
- 번역 / 감정 / 메모리 / 기타 작업에 대해 **별도 모델과 별도 파라미터**를 줄 수 있습니다.
- 슬롯 값은 글로벌보다 우선합니다.

---

## 4. 파라미터가 실제로 적용되는 순서

CPM에서 값이 겹치면 아래 순서로 적용됩니다.

1. **CPM 슬롯 오버라이드**
2. **RisuAI 파라미터 분리 값**
3. **RisuAI 메인 모델 파라미터**
4. **CPM 글로벌 기본값**

### 사용자가 체감하는 예시

#### 예시 1: 글로벌 온도만 넣은 경우
- RisuAI 쪽 온도가 비어 있으면 → **글로벌 온도**가 들어갑니다.
- RisuAI 쪽 온도가 이미 0.8이면 → **0.8이 그대로 사용**되고 글로벌 값은 무시됩니다.

즉, 글로벌 온도는
- “모든 요청을 0.7로 강제한다”가 아니라
- “아무 값도 없을 때 0.7을 대신 넣는다”에 가깝습니다.

#### 예시 2: 번역 슬롯 온도를 따로 넣은 경우
- 번역 슬롯에 온도 0.2를 넣으면
- 번역으로 판별된 요청은 **0.2가 우선**됩니다.
- 메인 채팅 온도나 글로벌 온도보다 슬롯 값이 먼저 적용됩니다.

#### 예시 3: 값을 비워둔 경우
- 해당 항목을 비워두면 CPM이 그 값을 건드리지 않습니다.
- 그래서 사용자는 “CPM을 켰는데도 원래랑 똑같다”라고 느낄 수 있는데,
  그건 정상입니다. **비워둔 항목은 원래 값이 그대로 지나갑니다.**

## 5. 프로바이더 탭별 설정 방법

아래는 **사용자 입장에서 꼭 만지는 항목만** 정리한 것입니다.

---

## 5-1. OpenAI

### 보통 꼭 넣는 것
- **API Key**: 필수

### 선택해서 쓰는 것
- **서버에서 모델 목록 불러오기**
  - 켜면 API에서 모델 목록을 읽어옵니다.
  - 보통 최신 모델을 편하게 고르고 싶을 때 사용합니다.
  - **체감 포인트:** 체크만 해서는 즉시 목록이 바뀌지 않을 수 있습니다. 보통 새로고침 후 모델 목록이 반영됩니다.
- **Reasoning Effort**
  - `o3`, `o1` 계열처럼 추론 모델에서 사용
  - 높일수록 더 생각하지만 느려지거나 비용이 늘 수 있습니다.
- **Response Verbosity**
  - 응답을 짧게/보통/길게 조절하는 느낌입니다.
- **Service Tier**
  - 응답 속도 우선 여부를 바꾸는 옵션입니다.
  - 잘 모르겠으면 `Auto` 권장
- **Prompt Cache Retention**
  - 프롬프트 캐시 관련 옵션
  - 잘 모르겠으면 기본값 유지 추천
- **Custom Base URL**
  - 호환 API, 프록시, 중계 서버를 쓸 때만 입력
  - 일반 OpenAI면 비워두면 됩니다.

### 여러 키를 넣고 싶을 때
- 여러 API 키를 **공백 / 줄바꿈**으로 구분해 넣으면 자동 키 회전이 됩니다.

---

## 5-2. Anthropic

### 보통 꼭 넣는 것
- **API Key**: 필수

### 자주 만지는 것
- **서버에서 모델 목록 불러오기**
  - 켜면 Claude 모델 목록을 가져옵니다.
- **Thinking Budget Tokens**
  - Claude 4.5 이하 계열에서 생각 토큰 예산
  - `0`이면 끔
- **Adaptive Thinking Effort**
  - 4.6 계열에서 추론 강도 조절
  - 잘 모르겠으면 비워두거나 기본값
- **Cache Enabled / Cache TTL**
  - 프롬프트 캐시 사용 여부
- **Custom Base URL**
  - 프록시/호환 서버가 있을 때만 입력

### 사용자가 느끼는 점
- Thinking 관련 값을 높이면 답이 더 신중해질 수 있지만, 속도와 사용량 부담이 생길 수 있습니다.
- 캐시를 켜면 반복 프롬프트에서 체감이 좋아질 수 있습니다.

---

## 5-3. Gemini Studio

### 보통 꼭 넣는 것
- **API Key**: 필수

### 자주 만지는 것
- **서버에서 모델 목록 불러오기**
- **Thinking Level**
  - Gemini 3용
  - 높일수록 더 적극적으로 생각하는 쪽
- **Thinking Budget Tokens**
  - Gemini 2.5용
  - `0`이면 끔
- **Preserve System**
  - 시스템 프롬프트를 보존할지 여부
- **Show Thoughts Token Info**
  - 생각 토큰 관련 알림 표시
- **Use Thought Signature**
  - 생각 서명 추출 사용
- **Use Plain Fetch**
  - 직접 요청 방식 사용
  - 프록시/V3 캐싱 우회가 필요할 때만 켜는 편이 좋습니다.

### 사용자가 느끼는 점
- Thinking Level / Budget을 올리면 더 오래 생각하는 느낌이 납니다.
- `Use Plain Fetch`는 문제 해결용 옵션에 가까워서, 특별한 이유 없으면 기본값 유지가 편합니다.

---

## 5-4. Vertex AI

### 보통 꼭 넣는 것
- **Service Account JSON Key**
  - 파일 경로가 아니라 **JSON 본문 전체**를 넣어야 합니다.
- **Location Endpoint**
  - 예: `global`, `us-central1`

### 자주 만지는 것
- **서버에서 모델 목록 불러오기**
- **Thinking Level / Thinking Budget Tokens**
  - Gemini 계열용
- **Claude Thinking Budget Tokens / Adaptive Thinking Effort**
  - Vertex의 Claude 계열용
- **Preserve System / Show Thoughts Token Info / Use Thought Signature**
  - Gemini/Claude 사용 방식에 따라 조정

### 여러 키를 넣고 싶을 때
- JSON 키는 **쉼표로 구분**해서 여러 개 넣을 수 있습니다.

### 사용자가 느끼는 점
- Vertex는 인증 자료를 잘못 넣으면 아예 모델 조회나 요청이 안 됩니다.
- 특히 JSON은 **파일 경로가 아니라 내용 전체**라는 점을 많이 헷갈립니다.

### 꼭 넣는 것

- **Secret Access Key**
- **Region**

### 선택 사항
- **서버에서 모델 목록 불러오기**

### 사용자가 느끼는 점

## 5-6. DeepSeek

### 꼭 넣는 것
- **API Key**

### 선택 사항

- **Custom Base URL**
  - 공식이 아닌 중계 엔드포인트를 쓸 때만 입력

### 여러 키를 넣고 싶을 때

## 5-7. OpenRouter

### 꼭 넣는 것
- **Model Name**
  - 예: `anthropic/claude-sonnet-4`
- **API Key**

### 선택 사항
- **Provider String**
  - 특정 공급자 라우팅을 강제하고 싶을 때만 입력
- **Reasoning Header**
- **Custom Base URL**

### 사용자가 느끼는 점
- OpenRouter는 다른 탭처럼 “모델 목록을 자동으로 받아와서 고르는 방식”보다,
  **모델 ID를 직접 적는 방식**으로 쓰는 느낌이 더 강합니다.
- 모델 이름 오타가 나면 바로 요청 실패로 이어집니다.

## 5-8. Copilot 탭이 포함된 경우

이 탭은 일반 API 키 입력 탭과 조금 다릅니다.

### 할 수 있는 것
- 토큰 직접 입력
- 토큰 생성
- 토큰 확인
- 모델 목록 확인
- 할당량 확인
- 자동 설정

### 처음 쓰는 사람 기준
- 이미 토큰이 있으면 **토큰 직접 입력 → 저장**
- 없으면 **토큰 생성**부터 진행
- 잘 되었는지 확인하려면 **토큰 확인** 또는 **모델 목록** 버튼 사용

## 6. 메인 모델 선택은 어디서 하나요?

중요한 점은:

- 프로바이더 탭은 **인증/옵션 설정용**이고
- 실제 채팅에 쓸 모델 선택은 보통 **RisuAI 메인 모델 선택 UI**에서 합니다.

즉,
1. CPM 안에서 키를 넣고
2. 필요하면 동적 모델 불러오기를 켜고
3. RisuAI의 모델 선택창에서 실제 모델을 고르는 흐름입니다.

### 모델 목록이 안 보일 때
아래 순서로 확인하면 됩니다.

1. 해당 프로바이더의 키가 제대로 들어갔는지 확인
2. 동적 모델 불러오기를 켰는지 확인
3. 새로고침 후 다시 확인
4. 그래도 안 뜨면 기본 제공 모델 또는 수동 모델명 입력 방식 사용

## 7. 글로벌 기본값 탭 사용법

탭 이름: **🎛️ 글로벌 기본값**

여기는 “모든 요청에 강제로 덮어쓰기”가 아니라,
**값이 비어 있을 때 넣는 기본값**이라고 생각하면 됩니다.

### 항목별 의미
- **Default Temperature**
  - 온도가 아예 없을 때만 대신 들어감
- **Default Max Output Tokens**
  - 최대 출력 토큰이 없을 때만 대신 들어감
- **Default Top P**
- **Default Frequency Penalty**
- **Default Presence Penalty**

### 이렇게 쓰면 좋음
- RisuAI 쪽 세팅을 자주 안 만지고, 전체 기본값만 대충 통일하고 싶을 때
- 여러 모델을 오가는데 “빈칸이면 최소한 이 정도로는 보내라”는 안전장치가 필요할 때

### 이렇게 생각하면 헷갈리지 않음
- **글로벌 = 보충값**
- **슬롯 = 강한 우선권**

---

## 8. 슬롯 탭 사용법

슬롯 탭은 아래 4개입니다.

- **번역 (Trans)**
- **감정 판독 (Emotion)**
- **하이파/메모리 (Mem)**
- **트리거/루아 (Other)**

각 탭에서 하는 일은 같습니다.

1. 그 작업을 맡길 **전담 모델 선택**
2. 필요하면 그 슬롯에만 적용할 **파라미터 별도 입력**

### 슬롯 모델을 지정하지 않으면
- 해당 작업도 그냥 메인 모델이 처리합니다.

### 슬롯 모델을 지정하면
- 해당 종류의 작업만 그 모델이 따로 처리합니다.

### 슬롯 파라미터를 입력하면
- 그 슬롯 요청에서는 해당 값이 우선합니다.

### 슬롯 파라미터를 비워두면
- CPM이 그 값을 건드리지 않습니다.
- 따라서 RisuAI 값 또는 글로벌 값이 그대로 사용됩니다.

---

## 8-1. 번역 슬롯

### 언제 쓰면 좋은가
- 메인 채팅은 큰 모델로 하고
- 번역만 빠르고 싼 모델에 맡기고 싶을 때

### 많이 쓰는 방식
- 번역 모델: 빠른 모델 지정
- Temperature: 낮게 (`0.1` ~ `0.3` 느낌)

### 사용자가 느끼는 점
- 번역은 온도를 낮게 두면 결과가 더 안정적인 편입니다.
- 번역 슬롯만 따로 두면 메인 채팅 품질과 번역 비용을 분리하기 좋습니다.

---

## 8-2. 감정 슬롯

### 언제 쓰면 좋은가
- 표정/반응 판독 같은 가벼운 보조 작업을 메인 모델과 분리하고 싶을 때

### 많이 쓰는 방식
- 작고 빠른 모델 배정
- 출력 토큰과 온도를 보수적으로 설정

### 사용자가 느끼는 점
- 메인 채팅이 무거워도 감정 판독이 덜 버벅이는 느낌을 받을 수 있습니다.

---

## 8-3. 메모리 슬롯

### 언제 쓰면 좋은가
- 긴 텍스트 요약, 메모리 압축 작업을 따로 돌리고 싶을 때

### 많이 쓰는 방식
- 요약에 강한 모델 지정
- Max Output Tokens를 적당히 제한

### 사용자가 느끼는 점
- 메모리/요약이 메인 대화 흐름과 분리되어 안정적일 수 있습니다.

---

## 8-4. Other 슬롯

### 언제 쓰면 좋은가
- Lua, 트리거, 무거운 유틸 요청을 별도 모델로 보내고 싶을 때

### 사용자가 느끼는 점
- 메인 대화와 기타 자동화 작업이 서로 덜 간섭합니다.

---

## 9. 슬롯 설정에서 가장 많이 헷갈리는 부분

### 메인 모델과 슬롯 모델을 같은 것으로 넣으면?
가능은 합니다. 하지만 **구분이 애매한 요청에서는 슬롯 오버라이드가 항상 확실하게 먹는다고 느껴지지 않을 수 있습니다.**

즉,
- 다른 모델을 슬롯에 넣으면 → 더 안정적으로 “분리해서 쓴다”는 느낌
- 같은 모델을 넣으면 → 상황에 따라 메인 값처럼 보일 수 있음

### 추천
- 슬롯 효과를 확실히 느끼고 싶으면 **메인 모델과 다른 모델**을 슬롯에 배정하는 편이 좋습니다.

## 10. 스트리밍 / 호환성 모드

탭 위치: **글로벌 기본값 → 스트리밍 설정**

### 스트리밍 패스스루 활성화
- 켜면 실시간 출력이 가능한 환경에서 더 자연스럽게 글이 흘러옵니다.
- 하지만 환경에 따라 자동으로 비활성화될 수 있습니다.

### Anthropic Thinking 토큰 표시
- Claude 계열의 thinking 관련 표시를 보고 싶을 때 사용

### 호환성 모드 강제 활성화
- iPhone / Safari / 일부 환경에서 요청이 꼬일 때 켜는 옵션
- **체감 포인트:** 켜면 스트리밍이 사실상 꺼지는 방향이라, 실시간 타이핑 느낌보다 안정성 우선이 됩니다.

### Node-less용 Copilot 실험 모드
- Copilot 전용 옵션입니다.
- Copilot이 특정 환경에서 안 될 때만 1번/2번을 바꿔가며 테스트하면 됩니다.
- 일반 사용자는 `끄기`로 두면 됩니다.

## 12. Sub-Plugins 탭 사용법

여기는 사용자가 배포받은 서브 플러그인을 관리하는 곳입니다.

### 할 수 있는 것
- 설치
- 켜기 / 끄기
- 삭제
- 업데이트 확인
- 전체 CPM 데이터 삭제

### 켜기 / 끄기
- 토글을 바꾸면 바로 반영됩니다.
- 기능이 이상할 때는 삭제보다 먼저 껐다 켜보는 편이 좋습니다.

### 업데이트 확인
- **업데이트 확인** 버튼을 누르면 새 버전이 있는지 확인합니다.
- 새 버전이 있으면 해당 버튼으로 바로 업데이트할 수 있습니다.

### 전체 CPM 데이터 삭제
- API 키, 슬롯 설정, 서브 플러그인 목록, 커스텀 모델까지 모두 지웁니다.
- 플러그인만 삭제해서는 데이터가 남을 수 있으므로,
  진짜 초기화가 필요할 때만 사용하세요.

## 13. Custom Models Manager

이 탭은 기본 제공 프로바이더가 아니라,
사용자가 **직접 호환 엔드포인트를 등록**하고 싶을 때 씁니다.

### 언제 쓰면 좋은가
- OpenAI 호환 API
- 자체 프록시 서버
- 공식 탭에 없는 엔드포인트

### 최소 입력만 하면 되는 항목
- **Display Name**
- **Model Name**
- **Base URL**
- **API Key**
- **API Format**

### 처음 쓰는 사람 팁
- 고급 옵션이 많아 보이지만, 처음에는 필수값만 넣고 저장해도 됩니다.
- 세부 플래그는 정말 필요한 경우에만 건드리는 편이 좋습니다.

### 중요한 점
- **추가/삭제 후에는 새로고침(F5)** 해야 네이티브 모델 목록에 반영될 수 있습니다.

- 장치 이동 전 한 번 더 내보내기

## 15. 초보자용 추천 세팅 순서

### 가장 무난한 시작법

1. 사용할 프로바이더 탭 하나만 먼저 설정
2. API 키 입력
3. 동적 모델 불러오기 필요하면 체크
4. 새로고침 후 메인 모델 선택
5. 글로벌 기본값은 일단 비워두기
6. 필요한 슬롯만 하나씩 추가

### 왜 이 순서가 좋은가
- 처음부터 글로벌/슬롯을 많이 건드리면 “어디서 바뀐 건지” 헷갈리기 쉽습니다.
- 먼저 **메인 모델 단독으로 정상 동작**을 확인한 뒤,
  글로벌 → 슬롯 순서로 추가하는 게 가장 덜 헷갈립니다.

## 16. 자주 겪는 상황별 안내

### 1) 글로벌 온도를 바꿨는데 체감이 없다
가능성이 큰 원인:
- RisuAI 쪽에서 이미 온도를 보내고 있음
- 슬롯 온도가 더 우선으로 걸려 있음

확인 방법:
- 슬롯 온도 비우기
- RisuAI 온도 확인
- 그래도 헷갈리면 API 보기로 실제 요청값 확인

### 2) 모델 목록이 안 늘어난다
가능성이 큰 원인:
- 키가 잘못됨
- 동적 모델 불러오기만 켜고 새로고침을 안 함
- 해당 프로바이더 API가 모델 목록 조회를 막음

### 3) 슬롯을 넣었는데 메인 모델처럼 보인다
가능성이 큰 원인:
- 슬롯 모델과 메인 모델이 같음
- 요청이 슬롯 작업으로 명확히 판별되지 않음

추천:
- 슬롯에 다른 모델을 배정해서 테스트

### 4) iPhone/Safari에서 이상하다
추천 순서:
1. 호환성 모드 켜기
2. 스트리밍 기대를 낮추기
3. Copilot이면 Node-less 실험 모드 변경 테스트

### 5) 설정이 너무 꼬였다
추천 순서:
1. 설정 내보내기로 백업
2. 불필요한 슬롯 값 정리
3. 필요하면 CPM 데이터 전체 삭제 후 재설정

## 17. 아주 짧은 요약

- **프로바이더 탭** = 키와 공급자 옵션 설정
- **메인 모델 선택** = RisuAI 쪽에서 실제 사용 모델 선택
- **글로벌 기본값** = 빈칸일 때만 들어가는 보충값
- **슬롯 설정** = 특정 작업에만 강하게 우선 적용되는 값
- **같은 모델을 메인/슬롯에 같이 넣으면** 분리 체감이 약할 수 있음
- **동적 모델 불러오기 / 커스텀 모델 추가 후에는 새로고침이 필요한 경우가 있음**
- **저장 버튼 없음, 바꾸는 즉시 저장**

## 18. 배포자용 안내 문구로 그대로 써도 되는 짧은 문장

아래 문장은 배포 글에 그대로 붙여 넣어도 무난합니다.

- 메인 플러그인 설치 후 CPM 설정창에서 서브 플러그인을 하나씩 추가 설치해주세요.
- 각 프로바이더 탭에서 API 키를 먼저 입력한 뒤, 필요하면 모델 목록 불러오기를 켜고 사용해주세요.
- 글로벌 기본값은 강제값이 아니라 빈칸 보충용입니다.
- 번역/감정/메모리/기타 슬롯은 메인 모델과 다른 모델을 넣어야 분리 체감이 가장 확실합니다.
- 설정은 저장 버튼 없이 즉시 저장됩니다.
