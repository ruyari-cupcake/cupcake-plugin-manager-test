## 1. CPM이 뭔가요?

**Cupcake Provider Manager (CPM)**는 RisuAI V3용 플러그인입니다.

하는 일은 간단합니다:
- **여러 AI 프로바이더**(OpenAI, Anthropic Claude, Google Gemini, Vertex AI, AWS Bedrock, DeepSeek, OpenRouter, GitHub Copilot)를 **한곳에서 관리**합니다.
- 각 프로바이더의 **API 키 입력, 모델 선택, 세부 설정**을 통합 설정 화면에서 처리합니다.
- 번역·감정·메모리 같은 **보조 작업을 별도 모델에 배정**할 수 있습니다.
- **키 회전**, **스트리밍**, **자동 업데이트** 등 편의 기능을 지원합니다.

> CPM은 RisuAI의 안전한 iframe 샌드박스 안에서 실행됩니다.  
> 브라우저 데이터나 시스템에 접근하지 않으며, 보안에 대한 자세한 내용은 README를 참고하세요.

## 2. 설치하기

### 2-1. 메인 플러그인 설치

1. **RisuAI**를 엽니다.
2. **설정 → 플러그인**으로 이동합니다.
3. **+ 버튼**을 눌러 배포된 `provider-manager.js` 파일을 선택합니다.
4. 설치가 완료되면 "Cupcake Provider Manager"가 플러그인 목록에 나타납니다.

### 2-2. 서브 플러그인 설치

메인 플러그인만으로는 아직 프로바이더가 등록되지 않습니다. **서브 플러그인**을 따로 설치해야 합니다.

1. CPM 설정창을 엽니다 (아래 2-3 참조).
2. 왼쪽 사이드바에서 **🧩 Sub-Plugins** 탭을 클릭합니다.
3. **설치할 서브 플러그인 선택** 영역을 클릭하여 `.js` 파일을 선택합니다.
4. 배포받은 서브 플러그인 파일들을 **하나씩** 설치합니다.

설치 가능한 서브 플러그인 종류:

| 파일명 | 이름 | 역할 |
|--------|------|------|
| `cpm-provider-openai.js` | 🟢 OpenAI | OpenAI GPT 계열 프로바이더 |
| `cpm-provider-anthropic.js` | 🟠 Anthropic | Claude 계열 프로바이더 |
| `cpm-provider-gemini.js` | 🔵 Gemini Studio | Google Gemini (API 키 방식) |
| `cpm-provider-vertex.js` | 🔷 Vertex AI | Google Vertex AI (서비스 계정 JSON) |
| `cpm-provider-aws.js` | 🔶 AWS Bedrock | AWS Bedrock Claude |
| `cpm-provider-deepseek.js` | 🟣 DeepSeek | DeepSeek Chat/Reasoner |
| `cpm-provider-openrouter.js` | 🌐 OpenRouter | OpenRouter 통합 |
| `cpm-copilot-manager.js` | 🔑 Copilot | GitHub Copilot 토큰 관리 |
| `cpm-chat-navigation.js` | 🧭 Chat Navigation | 채팅 메시지 이동 위젯 |
| `cpm-chat-resizer.js` | ↕️ Chat Input Resizer | 입력창 크기 조절기 |
| `cpm-translation-cache.js` | 💾 Translation Cache | 번역 캐시 관리자 |

> 💡 **모든 서브 플러그인을 다 설치할 필요는 없습니다.** 사용할 프로바이더와 기능만 골라서 설치하세요.

### 2-3. 설정창 여는 법

설치가 끝나면 아래 방법으로 CPM 설정창을 열 수 있습니다:

| 환경 | 방법 |
|------|------|
| PC | 키보드 단축키: **`Ctrl + Shift + Alt + P`** |
| 모바일 | 화면을 **4손가락으로 터치** |

설정창 상단에 `🧁 Cupcake PM v1.20.7` 같은 제목이 보이면 정상입니다.

## 3. 먼저 알아둘 것

### ⚡ 저장 버튼이 없습니다

- 설정값을 바꾸면 **그 순간 바로 저장**됩니다.
- 체크박스를 켜거나 끄면 즉시 반영됩니다.
- 실수로 바꿨다면 다시 원래 값으로 돌리면 됩니다.

### 📝 메인 모델 선택은 RisuAI에서

CPM 설정창은 **인증(키 입력)과 옵션 조정**용입니다.  
실제 채팅에 사용할 모델은 **RisuAI 메인 화면의 모델 선택창**에서 고릅니다.

흐름:
1. CPM에서 API 키 입력
2. 필요하면 "모델 목록 불러오기" 켜기
3. RisuAI 모델 선택창에서 원하는 모델 선택

## 4. 처음 설치 후 추천 순서

처음 설치했다면 아래 순서를 따르면 가장 편합니다:

1. 사용할 프로바이더 탭 하나만 먼저 열기
2. **API 키 입력**
3. **📡 서버에서 모델 목록 불러오기** 체크 (필요한 경우)
4. RisuAI 새로고침 (F5)
5. RisuAI 메인 모델 선택창에서 사용할 모델 선택
6. 글로벌 기본값은 일단 비워둠
7. 필요한 슬롯만 나중에 하나씩 추가

> **왜 이 순서인가요?**  
> 처음부터 글로벌값·슬롯을 많이 만지면 "어디서 값이 바뀐 건지" 헷갈립니다.  
> 먼저 메인 모델 단독으로 정상 동작을 확인한 뒤, 글로벌 → 슬롯 순으로 추가하는 게 가장 안전합니다.

## 6. 프로바이더 탭별 설정 방법

각 프로바이더 탭에서 설정하는 항목을 하나씩 설명합니다.  
**필수 항목**만 넣으면 바로 사용할 수 있고, 나머지는 필요할 때 조정하면 됩니다.

---

### 6-1. 🟢 OpenAI

| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| **API Key** | OpenAI API 키 (`sk-...`). 여러 개 넣으려면 공백 또는 줄바꿈으로 구분 | ✅ 필수 |
| **📡 서버에서 모델 목록 불러오기** | 체크하면 OpenAI API에서 최신 모델 목록을 자동으로 가져옴 | 선택 |
| **Reasoning Effort** | `o3`, `o1` 등 추론형 모델의 사고 강도. 높이면 더 신중하지만 느림/비쌈 | 선택 |
| **Response Verbosity** | 응답 길이 조절 (짧게 / 보통 / 길게) | 선택 |
| **Service Tier** | 응답 속도 우선 여부. 잘 모르면 `Auto` | 선택 |
| **Prompt Cache Retention** | 프롬프트 캐시 유지 시간. 잘 모르면 기본값 | 선택 |
| **Custom Base URL** | 프록시나 호환 API 서버 주소. 일반 OpenAI면 **비워두기** | 선택 |

**Reasoning Effort 옵션:**
- `None (없음)` — 미설정
- `Off (끄기)` — 끔
- `Low (낮음)` / `Medium (중간)` / `High (높음)` / `XHigh (매우 높음)`

**Service Tier 옵션:**
- `Auto (자동)` — 추천
- `Flex` — 비용 최적화
- `Default`

**Prompt Cache Retention 옵션:**
- `None (기본, 서버 자동 5~10분)`
- `In-Memory (5~10분, 최대 1시간)`
- `24h Extended (24시간 확장 캐시)`

---

### 6-2. 🟠 Anthropic

| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| **API Key** | Anthropic API 키. 여러 개는 공백/줄바꿈 구분 | ✅ 필수 |
| **📡 서버에서 모델 목록 불러오기** | Claude 모델 목록 자동 가져오기 | 선택 |
| **Thinking Budget Tokens** | Claude 4.5 이하 모델의 사고 토큰 예산. `0`이면 끔 | 선택 |
| **Adaptive Thinking Effort** | Claude 4.6 모델용 추론 강도 (`low` / `medium` / `high` / `max`) | 선택 |
| **Cache Enabled** | 프롬프트 캐싱 사용 여부 | 선택 |
| **Cache TTL** | 캐시 유지 시간 | 선택 |
| **Custom Base URL** | 프록시/호환 서버가 있을 때만 입력 | 선택 |

**Cache TTL 옵션:**
- `Default (5분, GA)`
- `Extended 1시간 (Beta)`

**사용자 팁:**
- Thinking 값을 높이면 답이 더 신중해지지만, 속도가 느려지고 비용이 늘 수 있습니다.
- 캐시를 켜면 반복 프롬프트에서 비용이 줄어들 수 있습니다.

---

### 6-3. 🔵 Gemini Studio

| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| **API Key** | Google AI Studio API 키. 여러 개는 공백/줄바꿈 구분 | ✅ 필수 |
| **📡 서버에서 모델 목록 불러오기** | Gemini 모델 목록 자동 가져오기 | 선택 |
| **Thinking Level** | Gemini 3용 사고 수준 | 선택 |
| **Thinking Budget Tokens** | Gemini 2.5용 사고 토큰 예산. `0`이면 끔 | 선택 |
| **Preserve System** | 시스템 프롬프트를 보존할지 여부 | 선택 |
| **Show Thoughts Token Info** | 사고 토큰 관련 알림 표시 | 선택 |
| **Use Thought Signature** | 사고 서명 추출 사용 | 선택 |
| **Use Plain Fetch** | 직접 요청 방식 사용. 문제 해결용 옵션 | 선택 |

**Thinking Level 옵션:**
- `Off (끄기)` / `None (없음)` / `Minimal (최소)` / `Low (낮음)` / `Medium (중간)` / `High (높음)`

**사용자 팁:**
- Thinking Level/Budget을 올리면 더 오래 생각하는 느낌이 납니다.
- `Use Plain Fetch`는 특별한 문제가 아니면 기본값(꺼짐) 유지가 좋습니다.

---

### 6-4. 🔷 Vertex AI

| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| **Service Account JSON Key** | 서비스 계정의 **JSON 본문 전체**를 넣어야 함 (파일 경로 ❌) | ✅ 필수 |
| **Location Endpoint** | 리전 (예: `global`, `us-central1`) | ✅ 필수 |
| **📡 서버에서 모델 목록 불러오기** | 모델 목록 자동 가져오기 | 선택 |
| **Thinking Level** | Gemini 3용 사고 수준 | 선택 |
| **Thinking Budget Tokens** | Gemini 2.5용 사고 토큰 예산 | 선택 |

**Claude on Vertex (Model Garden) 섹션:**

Vertex AI에서는 Gemini뿐만 아니라 **Claude 모델**도 사용할 수 있습니다.

| 항목 | 설명 |
|------|------|
| **Claude Thinking Budget Tokens** | Claude 4.5 이하 모델용 사고 예산. `0`이면 끔 |
| **Adaptive Thinking Effort** | Claude 4.6 모델용 추론 강도 |

**공통 옵션:**
| 항목 | 설명 |

**여러 키 입력:**
- JSON 키를 **쉼표로 구분**해서 여러 개 넣을 수 있습니다.

> ⚠️ **가장 많이 하는 실수:** JSON은 **파일 경로**가 아니라 `{"type": "service_account", ...}` 형태의 **내용 전체**를 붙여넣어야 합니다.

## 7. 글로벌 기본값 설정

탭: **🎛️ 글로벌 기본값 (Global Fallback Parameters)**

### 이 탭의 핵심 개념

> **글로벌 기본값은 "강제 덮어쓰기"가 아니라, RisuAI가 값을 안 보냈을 때만 대신 들어가는 "빈칸 보충값"입니다.**

- RisuAI 쪽에서 이미 온도가 0.8로 설정되어 있으면 → 글로벌 기본값은 **무시**됩니다.
- RisuAI 쪽 온도가 비어 있으면 → 글로벌 기본값이 **대신 들어갑니다**.
- 비워두면 CPM이 그 항목을 건드리지 않고 원래 값이 그대로 전송됩니다.

### 설정 항목

| 항목 | 설명 |
|------|------|
| **Default Temperature** | 기본 온도 (비워두면 미전송) |
| **Default Max Output Tokens** | 기본 최대 출력 토큰 (비워두면 미전송) |
| **Default Top P** | 기본 Top P (비워두면 API 기본값) |
| **Default Frequency Penalty** | 기본 빈도 페널티 (비워두면 API 기본값) |
| **Default Presence Penalty** | 기본 존재 페널티 (비워두면 API 기본값) |

### 이렇게 쓰면 좋습니다

- 여러 모델을 오가면서 **최소한 이 정도는 보내라**는 안전장치가 필요할 때
- RisuAI 쪽 세팅을 자주 건드리지 않고 **전체 기본값만 대충 통일**하고 싶을 때
- 잘 모르겠으면 **비워두는 게 가장 안전**합니다

## 8. 보조 슬롯 설정 (Aux Slots)

보조 슬롯은 **번역·감정·메모리·기타** 4가지 작업을 **메인 채팅과 다른 모델**에 맡길 수 있는 기능입니다.

### 왜 쓰나요?

- 메인 채팅은 큰 고급 모델로 하고
- 번역 같은 보조 작업은 빠르고 저렴한 모델에 맡기면
- **비용은 줄이면서** 메인 채팅 품질은 유지할 수 있습니다.

### 공통 설정 방법

모든 슬롯 탭의 구조는 동일합니다:

1. **전담 모델 선택** — 드롭다운에서 모델을 고릅니다.
   - `🚫 미지정`으로 두면 메인 모델이 그대로 처리합니다.
2. **Generation Parameters (생성 설정)** — 해당 슬롯에만 적용할 파라미터를 별도로 입력합니다.
   - 비워두면 CPM이 건드리지 않고 RisuAI 값 또는 글로벌 값이 그대로 사용됩니다.

#### 생성 설정 항목 (모든 슬롯 공통)

| 항목 | 설명 |
|------|------|
| **Max Context Tokens** | 최대 컨텍스트 |
| **Max Output Tokens** | 최대 응답 크기 |
| **Temperature** | 온도 |
| **Top P** | 오답 컷팅 |
| **Top K** | 오답 컷팅 |
| **Repetition Penalty** | 반복 페널티 |
| **Frequency Penalty** | 빈도 페널티 |
| **Presence Penalty** | 존재 페널티 |

---

### 8-1. 🌐 번역 슬롯

**탭:** 🌐 번역 (Trans)  
**설명:** 번역 태스크만 자동으로 가져와서 전담할 프로바이더를 선택합니다.

**추천 설정:**
- 번역 모델: 빠르고 가벼운 모델 지정
- Temperature: 낮게 (`0.1` ~ `0.3`) → 번역 결과가 더 안정적

**체감:**
- 번역 슬롯만 따로 두면 메인 채팅 품질과 번역 비용을 분리하기 좋습니다.

---

### 8-2. 😊 감정 판독 슬롯

**탭:** 😊 감정 판독 (Emotion)  
**설명:** 캐릭터 리액션/표정 태스크를 담당할 작고 빠른 모델을 지정합니다.

**추천 설정:**
- 작고 빠른 모델 배정
- 출력 토큰과 온도를 보수적으로 설정

**체감:**
- 메인 채팅이 무거워도 감정 판독이 덜 버벅이는 느낌을 받을 수 있습니다.

---

### 8-3. 🧠 메모리(하이파) 슬롯

**탭:** 🧠 하이파 (Mem)  
**설명:** 채팅 메모리 요약 등 긴 텍스트 축약 역할을 전담할 모델을 지정합니다.

**추천 설정:**
- 요약에 강한 모델 지정
- Max Output Tokens를 적당히 제한

**체감:**
- 메모리/요약이 메인 대화 흐름과 분리되어 더 안정적일 수 있습니다.

---

### 8-4. ⚙️ 트리거/루아 슬롯

**탭:** ⚙️ 트리거/루아 (Other)  
**설명:** Lua 스크립트, 트리거, 무거운 유틸 요청을 별도 모델로 보냅니다.

**체감:**
- 메인 대화와 기타 자동화 작업이 서로 덜 간섭합니다.

---

### 8-5. 파라미터 적용 우선순위

값이 겹칠 때 아래 순서로 적용됩니다 (위가 강함):

| 우선순위 | 출처 | 설명 |
|----------|------|------|
| 1순위 | **CPM 슬롯 오버라이드** | 슬롯에 별도 입력한 값 |
| 2순위 | **RisuAI 파라미터 분리 값** | RisuAI에서 프로바이더별로 분리 설정한 값 |
| 3순위 | **RisuAI 메인 모델 파라미터** | RisuAI 메인 화면의 파라미터 |
| 4순위 | **CPM 글로벌 기본값** | 위 세 곳이 모두 비어 있을 때만 |

**예시:**
- 번역 슬롯에 온도 0.2를 넣으면 → 번역 요청은 무조건 0.2가 적용.
- RisuAI에서 온도 0.8을 설정하면 → 글로벌 기본값 0.7은 무시됨.
- 모든 곳을 비워두면 → API 쪽 기본값이 그대로 사용됨.

> 💡 **팁:** 슬롯 효과를 확실히 느끼려면 **메인 모델과 다른 모델**을 슬롯에 배정하세요.  
> 같은 모델을 넣으면 분리 체감이 약할 수 있습니다.

## 11. 커스텀 모델 매니저

탭: **🛠️ Custom Models Manager**

기본 제공 프로바이더 탭에 없는 **호환 엔드포인트를 직접 등록**하고 싶을 때 사용합니다.

### 언제 쓰나요?

- OpenAI 호환 API 서버를 쓸 때
- 자체 프록시 서버를 연결할 때
- 공식 탭에 없는 엔드포인트를 추가할 때

### 모델 추가 방법

1. **➕ Add Model** 버튼을 클릭합니다.
2. 아래 항목을 입력합니다.

#### 필수 입력 항목

| 항목 | 설명 |
|------|------|
| **Display Name** | 모델 표시 이름 (내가 알아볼 이름) |
| **Model Name** | API에서 사용하는 모델 ID |
| **Base URL** | API 엔드포인트 주소 |
| **API Key** | API 키 (여러 개 입력 시 공백/줄바꿈 구분 → 자동 키 회전) |
| **API Format** | 포맷 선택: `OpenAI` / `Anthropic Claude` / `Google Gemini` |

#### 선택 입력 항목

| 항목 | 설명 |
|------|------|
| **CORS Proxy URL** | Copilot Node-less CORS 우회용 프록시 URL |
| **Tokenizer** | 토크나이저 종류: `o200k_base` / `llama3` / `Claude` / `Gemma` |
| **Responses API Mode** | `Auto` / `On` / `Off` |
| **Thinking Level** | `Off` / `None` / `Minimal` / `Low` / `Medium` / `High` |
| **Thinking Budget Tokens** | 사고 토큰 예산 (`0` = 끔) |
| **Max Output Tokens** | 최대 출력 토큰 (`0` = 무제한) |
| **Prompt Cache Retention** | `None` / `In-Memory` / `24h Extended` |
| **Reasoning Effort** | `None` / `Off` / `Low` / `Medium` / `High` / `XHigh` |
| **Response Verbosity** | `None` / `Low` / `Medium` / `High` |
| **Anthropic Effort** | `사용 안함` / `미지정` / `Low` / `Medium` / `High` / `Max` |

#### 고급 플래그 (체크박스)

| 플래그 | 설명 |
|--------|------|
| **hasFirstSystemPrompt** | 첫 시스템 프롬프트 유지 |
| **mergeSystemPrompt** | 시스템 프롬프트 병합 |
| **requiresAlternateRole** | 교대 역할 필수 |
| **mustStartWithUserInput** | 유저 메시지로 시작 필수 |
| **useMaxOutputTokensInstead** | max_output_tokens 사용 |
| **Use Streaming** | 스트리밍 사용 |
| **useThoughtSignature** | 사고 서명 사용 |
| **useAdaptiveThinking** | 적응형 사고 사용 |

#### Custom Parameters JSON

- 자유 형식 JSON 텍스트 영역
- API 요청에 추가할 커스텀 파라미터를 직접 지정할 수 있습니다

### 모델 관리

| 버튼 | 설명 |
|------|------|
| ✏️ 편집 | 모델 설정 수정 |
| 🗑️ 삭제 | 모델 삭제 (확인 필요) |
| 📤 내보내기 | 개별 모델을 `.cpm-model.json`으로 저장 |
| 📥 Import Model | `.json` 파일에서 모델 가져오기 (다중 파일 지원) |

> ⚠️ **중요:** 모델을 추가하거나 삭제한 후에는 **RisuAI를 새로고침(F5)**해야 네이티브 모델 목록에 반영됩니다.

> 💡 **처음 쓰는 사람 팁:** 고급 옵션이 많아 보이지만, 처음에는 **필수 항목 5가지만** 넣고 저장해도 됩니다.

## 15. 번역 캐시 관리자 (Translation Cache)

서브 플러그인: **`cpm-translation-cache.js`**  
탭: **💾 번역 캐시**

RisuAI의 LLM 번역 캐시를 검색·확인하고, **사용자 수정 사전**으로 번역을 교정합니다.

### 기본 개념

- RisuAI의 번역 캐시 자체는 **읽기 전용**입니다.
- 번역을 수정하면 **수정 사전**에 별도 저장되고, 채팅 표시 시점에 자동으로 적용됩니다.

### 설정 항목

| 항목 | 설명 |
|------|------|
| **수정 사전 자동 적용** | 채팅 표시 시 번역 교정을 자동 적용할지 여부 |

### 통계 표시

- `RisuAI 번역 캐시: N건` — 전체 캐시 건수 (🔄 새로고침 가능)
- `사용자 수정 사전: N건` — 사용자가 수정한 번역 건수

### 검색

- 검색창에 입력하면 **원문 + 번역문 모두** 검색됩니다.

### 기능 버튼

| 버튼 | 설명 |
|------|------|
| 📋 **캐시 전체 보기** | 전체 번역 캐시를 목록으로 봅니다 |
| 📝 **수정 사전 보기** | 사용자가 수정한 번역 목록을 봅니다 |
| ➕ **수동 추가** | 수정 사전에 새 항목을 직접 추가합니다 |
| 📤 **전체 내보내기** | 캐시 + 수정 사전을 합친 JSON을 저장합니다 |
| 📤 **수정 사전 내보내기** | 수정 사전만 JSON으로 저장합니다 |
| 📥 **가져오기** | JSON 파일에서 번역 데이터를 가져옵니다 |
| 🗑️ **수정 사전 전체 삭제** | 수정 사전을 모두 지웁니다 |

### 개별 항목 관리

번역 항목을 열면:
- 🔍 **상세** — 원문/번역문 상세 확인
- ✏️ **수정** — 번역문을 직접 수정
- ↩️ **수정 되돌리기** — 수정한 번역을 원래대로 복원

## 18. 프록시 서버 사용하기

CPM은 여러 경로로 프록시 서버를 활용할 수 있습니다.

### 18-1. Vercel 기반 업데이트 서버 (기본)

**사용자가 별도로 설정할 필요 없습니다.** CPM의 자동 업데이트, 버전 확인, 서브 플러그인 업데이트는 모두 Vercel에 배포된 서버를 통해 자동으로 이루어집니다.

| 기능 | 설명 |
|------|------|
| 버전 확인 | 5분 캐시로 최신 버전 유무를 자동 확인 |
| 메인 플러그인 업데이트 | 새 버전이 있으면 자동 알림 |
| 서브 플러그인 업데이트 | 설정창에서 업데이트 확인 버튼으로 확인 가능 |

### 18-2. Oracle 프록시 서버 (선택)

> ⚠️ 이 기능은 **아직 설계 단계**이며, 배포되지 않았습니다. 향후 지원 예정입니다.

Oracle VM 기반 프록시 서버로, Copilot이나 Anthropic API의 CORS 제한을 우회하기 위한 중계 서버입니다.

**사용자 관점에서:**
- 관리자가 프록시 서버를 배포한 경우, **Custom Base URL**에 프록시 서버 주소를 입력합니다.
- 인증은 관리자로부터 받은 **공유 비밀키 (Shared Secret)**를 사용합니다.

**프록시 서버 엔드포인트:**
| 경로 | 설명 |
|------|------|
| `GET /health` | 서버 상태 확인 |
| `GET /v1/models` | 사용 가능 모델 목록 |
| `POST /v1/messages` | AI 메시지 프록시 |

**인증 방법** (둘 중 하나):
- 헤더: `Authorization: Bearer <공유비밀키>`
- 헤더: `x-proxy-secret: <공유비밀키>`

> 💡 프록시 서버는 요청/응답 본문을 저장하지 않으며, API 키 등 민감 정보는 항상 마스킹 처리됩니다.

### 18-3. Custom Base URL로 프록시 연결

각 프로바이더 탭에는 **Custom Base URL** 입력란이 있습니다. 이것을 사용하면:

1. 공식 API 서버 대신 **프록시나 호환 서버**로 요청을 보낼 수 있습니다.
2. 프록시 서버 주소를 입력하면 됩니다.

**예시:**
- OpenAI 호환 서버: `https://my-proxy.example.com/v1`
- Anthropic 프록시: `https://proxy.example.com`

**사용하지 않을 때는 비워두세요** — 비워두면 각 프로바이더의 공식 엔드포인트로 직접 요청됩니다.

또한 **커스텀 모델 매니저**에서도:
- **Base URL** 필드에 프록시 주소를 넣을 수 있습니다.
- **CORS Proxy URL** 필드에 별도의 CORS 우회 프록시를 지정할 수 있습니다.

## 19. 키 회전 (Key Rotation)

CPM에서는 **여러 API 키를 동시에 등록**하면 자동으로 키를 돌려가며 사용합니다.

### 사용법

키 입력란에 여러 키를 넣을 때:

| 프로바이더 | 구분 방법 |
|-----------|-----------|
| OpenAI, Anthropic, Gemini, DeepSeek, OpenRouter | **공백** 또는 **줄바꿈**으로 구분 |
| Vertex AI (JSON) | **쉼표**로 구분 |
| 커스텀 모델 | **공백** 또는 **줄바꿈**으로 구분 |

### 동작 방식

1. 요청마다 등록된 키 중 **랜덤으로 하나**를 선택합니다.
2. 해당 키가 **429 (요청 한도)**, **529**, **503 (서비스 불가)** 에러를 받으면:
   - 그 키를 잠시 제외하고 **다른 키로 재시도**합니다.
3. 모든 키가 실패하면 **30초 대기** 후 다시 시도합니다.
4. 최대 **30번**까지 재시도합니다.

### 이렇게 쓰면 좋습니다

- API 키가 2~3개 있을 때 모두 넣어두면 **한도 분산** 효과
- 한 키가 한도에 걸려도 다른 키로 자동 전환되어 **끊김 최소화**

## 20. 자주 묻는 질문 (FAQ)

### Q: 글로벌 온도를 바꿨는데 체감이 없어요

**가능한 원인:**
- RisuAI에서 이미 온도를 설정하고 있음 (글로벌보다 RisuAI 값이 우선)
- 슬롯 온도가 걸려 있음 (슬롯이 가장 우선)

**확인 방법:**
- 슬롯 온도 비우기
- RisuAI 온도 확인
- **📡 API 보기**로 실제 전송값 확인

---

### Q: 모델 목록이 안 늘어나요

**가능한 원인:**
- 키가 잘못됨
- "모델 목록 불러오기"만 켜고 새로고침을 안 함
- 해당 API가 모델 조회를 제한함

**해결 순서:**
1. API 키가 맞는지 확인
2. 📡 서버에서 모델 목록 불러오기 체크
3. RisuAI **새로고침 (F5)**
4. 그래도 안 뜨면 수동으로 모델명 입력 또는 커스텀 모델 사용

---

### Q: 슬롯을 넣었는데 메인 모델처럼 보여요

**가능한 원인:**
- 슬롯 모델과 메인 모델이 **동일 모델**
- 요청이 해당 슬롯 작업으로 명확히 판별되지 않음

**해결:**
- 슬롯에 **다른 모델**을 배정해서 테스트

---

### Q: iPhone/Safari에서 이상해요

**추천 순서:**
1. **호환성 모드 켜기** (글로벌 기본값 탭)
2. 스트리밍이 꺼지는 건 정상 — 안정성 우선
3. Copilot이면 **Node-less 실험 모드** 1, 2번을 바꿔가며 테스트

---

### Q: 설정이 너무 꼬였어요

**추천 순서:**
1. **⬇️ 설정 내보내기**로 현재 상태 백업
2. 불필요한 슬롯 값 비우기
3. 그래도 안 되면 Sub-Plugins 탭의 **전체 CPM 데이터 삭제** → 처음부터 재설정

---

### Q: 서브 플러그인 업데이트는 어떻게 하나요?

1. CPM 설정창 → 🧩 Sub-Plugins 탭
2. **🔄 서브 플러그인 업데이트 확인** 버튼 클릭
3. 업데이트가 있으면 해당 서브 플러그인 옆에 업데이트 버튼이 표시됩니다
4. 버튼을 누르면 바로 업데이트됩니다

---

### Q: Custom Base URL에 뭘 넣어야 하나요?

- **일반적인 경우**: 비워두세요. 프로바이더 공식 서버로 직접 연결됩니다.
- **프록시 서버가 있는 경우**: 프록시 서버 주소를 입력합니다 (예: `https://proxy.example.com/v1`)
- **호환 API를 쓰는 경우**: 해당 서버의 API 주소를 입력합니다
